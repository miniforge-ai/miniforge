## Theme — DAG orchestration (`dag-orchestration`, status: in-flight)

Miniforge's DAG executor exists and is wired but never fires for
   LLM-generated plans because `plan-from-agent` output doesn't reach
   the DAG check with a :plan/id. Unlocking this turns every non-trivial
   spec from a monolithic implement phase into parallel child workflows
   — the single biggest leverage point for dogfood efficiency.

**Informative spec:** `specs/informative/I-DAG-ORCHESTRATION.md`
